Prerequisites

In order to test this example, following components are required.

Play Framework

You should download and install Play Framework version 2.2.0 or greater (direct download link).

You are also encouraged to read some documentation and example from http://www.playframework.com.

Bootstrap (optional)

Bootstrap is a front-end framework. It is used to generate pretty HTML pages with little effort.

Necessary files can be downloaded and put to appropriate place. Then meta.scala.html should be updated accordingly.

Or, you can keep files unchanged and still point to external location.

Antidot Java API

Yes, you need it! You can configure your Play project by adding following lines to your build.sbt

libraryDependencies += "net.antidot" % "afs-api" % "1.1"

Remember to leave at least one empty line before and after this declaration.

Antidot search service

For purpose of this example you can refer free available service with id 70000 at eval.partners.antidot.net.

Quick Play environment

For those of you who want to test Play framework almost immediately, you can get all source code from https://github.com/antidot/Play.

Set up example step by step

Most of the steps to set up this example are detailed in AFS search query coder/decoder API, AFS search query API and AFS search reply API.

Here focus is given on missing steps in order to make the example fully functional.

Initialization

It is highly advised to modify default Play application example to setup this example. Moreover, you should use powerfull Java IDE such as Eclipse to simplify import of required classes.

If you still want to do it by hand, here are the necessary import for the full example:

import net.antidot.api.common.Service;
import net.antidot.api.search.CoderManager;
import net.antidot.api.search.Connector;
import net.antidot.api.search.Facet;

import net.antidot.api.search.QueryManager;
import net.antidot.api.search.RepliesHelper;
import net.antidot.protobuf.reply.Reply.replies;

Generate HTML page

This is the last step.
Views are compiled and available from Java code. To render the page you simple have to call the render method on the appropriate view:

return ok(meta.render(helper));

Java view examples

They are available here: https://github.com/antidot/Java_API/tree/master/doc/data/views. But for convenience, content of each view file is given below.

These views are part of the Play project. If necessary, more documentation is available on Play website.

Main view

This view is called to render the HTML page.

<!-- HTML5 shim and Respond.js IE8 support of HTML5 elements and media queries -->

<![endif]-->
</head>
<body>
<div class="page-header">
<h1>Example <small>based on the Antidot Java API</small></h1>
</div>
@if(replies.hasReplySet()) {
@defining(replies.getReplySet()) { replySet =>

<div class="col-md-5"></div>

Here, some checks are performed to verify that there is a reply and whether the reply contains a pager. Except that it refers three views, there is nothing else noticable here.

Refered views are described below.

Facet view

facet.scala.html

@(facets: List[net.antidot.api.search.FacetHelper])
@if(facets.size() > 0) {
<div class="col-md-3">
<h2>Filters</h2>
@for(facet <- facets) {
<div class="panel panel-default">
<div class="panel-heading">@facet.getLabel()</div>
<div class="panel-body">
@if(facet.getValues().size() > 0) {
<div class="list-group">
@facetValues(facet)
</div>
}
</div>
</div>
}
</div>
}

It is quite easy to find the loop which allows to set up all facets. Then, in turn, this view refers specific view for facet values.

facetValues.scala.html

@(facet: net.antidot.api.search.FacetHelper)
<div class="list-group">
@for(value <- facet.getValues()) {
@defining(if(facet.isSet(value.getKey())) "active" else "") { active =>
<a class="list-group-item @active" href="@facet.getLink(value.getKey())">@value.getLabel()<span class="badge">@value.getCount()</span></a>
}
}
</div>

Here again, the loop allows to set up all values for each facet.

Result view

This view allows to render URI, title and abstract of each result. Some additional work is necessary if you want to get access to client data.

result.scala.html

@(meta: net.antidot.api.search.MetaHelper, replies: List[net.antidot.api.search.ReplyHelper])
<div class="col-md-9">
<div class="row">
<div class="col-md-1"></div>
<div class="col-md-4">
<h2>Results <span class="label label-success">@meta.getReplyNb()</span></h2>
<h4><span class="label label-info">Duration @meta.getDuration() ms</span></h4>
</div>
</div>
<ul class="list-unstyled">
@for(reply <- replies) {
<li>
<h3>@Html(reply.getTitle())</h3>
<p><a href="@reply.getUri()">@reply.getUri()</a></p>
<p>@Html(reply.getAbstract())</p>
</li>
}
</ul>
</div>

Pager view

This view generates simple pager at the bottom of the page. There is really no difficulty here.

pager.scala.html

@(pager: net.antidot.api.search.PagerHelper)
<div class="row">
<div class="col-md-5"></div>
<div class="input-group col-md-3">
<ul class="pagination">
@if(pager.hasPrevious()) {
<li><a href="@pager.getLink(pager.getPrevious())">Previous</a></li>
}
@for(page <- pager.getPages()) {
@defining(if(pager.getCurrent() == page) "active" else "") { active =>
<li class="@active"><a href="@pager.getLink(page)">@page</a></li>
}
}
@if(pager.hasNext()) {
<li><a href="@pager.getLink(pager.getNext())">Next</a></li>
}
</ul>
</div>
</div>

Example conclusion

Provided example is composed of about 30 lines of Java code. About third of these lines are related to facet configuration. So in few Java code lines and some scala views, you can get a fully functional Web interface for your Antidot search service.
